# 题意

给你一个字符串 sss ，由 nnn 个字符 0 和 / 或 1 组成。

你复制了这个字符串的 mmm 个副本，让第 iii 个副本成为字符串 tit_iti​。然后在每个副本上执行一个操作：对于 iii-th 副本，对其子串 [li,ri][l_i, r_i][li​,ri​]（从 lil_ili​-th 字符到 rir_iri​-th 字符的子串，包括两个端点）进行排序。注意：每个操作只影响一个副本，每个副本只受一个操作的影响。

你的任务是计算t1,t2,…,tmt_1, t_2, \ldots, t_mt1​,t2​,…,tm​ 中不同字符串的数量。请注意，只有在操作后至少有一个副本保持不变时，才应计算初始字符串 sss。

# 思路

要统计有多少个不同的字符串，即统计有多少个不同的区间，但不同的区间可能会生成同一个字符串，这是就需要我们对区间进行处理，找到区间的临界值，再将这个临界值放到 set 集合中进行去重。
如果s[l]s[l]s[l] 是000 的话，我们可以将lll 不断地加加，找到第一个s[l]s[l]s[l] 等于111 的地方。同理，如果我们s[r]s[r]s[r] 是111 的话，可以将rrr 不断地减减，找到第一个s[r]s[r]s[r] 等于000 的地方，这样得到的就是临界值的[l,r][l, r][l,r]，举个例子假设 s="000011", 区间[1,6][1,6][1,6] 和区间[4,5][4,5][4,5] 得到就是同一个字符串。
但是这样得到lll 可能会大于rrr, 因为它排序得到的字符串会是原字符串，所以我们只需要将临界值修改为 [-1,-1] 就行。
查找第一个 1 的位置和第一个 0 的位置，可以使用到 lower_bound 这个函数进行二分找到它们的位置。
